## The architectural milestone
- **First cross-module *call*.** Previous modules only *read* across boundaries (via batch
  getters). `offers.acceptOffer` **invokes** `deals.advanceDealStage` — module calling module,
  through the public interface, never the tables. Dependency stays one-way (`offers → deals,
  contacts`); nothing depends back on `offers`; no cycles. The O-06 boundary test passes.
- Accept advances the stage *first*, then marks the offer accepted, so accepting on a settled deal
  aborts cleanly with no half-applied state.
