## Architectural notes (for sanity-check)
- **Module boundaries held across four modules.** `deals` needs property + contact data for its
  board/detail but does **no cross-module SQL joins** (rule 3): its repository only queries deals'
  own tables; property/contact data is hydrated via the `properties`/`contacts` interfaces
  (`getPropertiesByIds` / `getContactsByIds`). Dependencies stay one-way; no cycles.
- **`stage_history` is append-only at the database level** — written only by the SECURITY DEFINER
  trigger; INSERT/UPDATE/DELETE revoked from the API roles. Integration test proves the app role's
  delete/update is rejected (D-07).
- Migrations `001–005`, all applied to cloud.
- Learned + logged: a **Client Component must not import a *value* from a module barrel** (it drags
  in `next/headers`); pass such values as props. The build — not the tests — catches it.
