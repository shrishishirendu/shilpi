## ⚠️ Two latent schema bugs the integration tests caught (please sanity-check the fixes)
Both were in the original schema and would have surfaced painfully in production. This is the
direct payoff of testing against a real Postgres instead of mocks:

1. **Missing DML grants → `003_grants.sql`.** A hosted Supabase project grants
   SELECT/INSERT/UPDATE/DELETE to the API roles implicitly; a local `supabase start` did not, so
   even `service_role` hit `permission denied`. Fix: explicit grants. **RLS still fully enforces
   tenancy — grants only decide table *reachability*.**
2. **RLS infinite recursion → `004`.** `current_agency_id()` reads `users`, whose RLS policy
   calls `current_agency_id()` → infinite recursion → `stack depth limit exceeded` on *any*
   authenticated query. Fix: make the helper `SECURITY DEFINER` so it reads `users` without
   re-entering RLS (standard Supabase pattern). Tenancy unchanged.

**No table-structure drift** — both are grants/function-security only. Flagged because they
touch the security model.
