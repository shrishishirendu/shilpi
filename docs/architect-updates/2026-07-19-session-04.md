## Decisions / notes
- **Module boundary pattern is now set** by `contacts`: owns its table, depends only on
  `platform`, exposes a narrow `index.ts`; the app imports only `@/modules/contacts`, never
  `internal/*`. `properties` and `deals` will follow the same shape.
- **D2 (unified contacts) is live**: a contact is one role-agnostic person. Roles will attach via
  `deal_contacts` when the `deals` module is built. (The human asked how to interpret a contact ‚Äî
  answer: it's a person the agency deals with (buyer/vendor/solicitor), *not* a login; logins are
  `users`.)
- Local dev + tests run against local Supabase; production uses the cloud project's env.
